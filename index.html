Introducción a la calidad de desarrollo de software. Principalmente en este parcial vemos los diferentes conceptos y características de la calidad de software. Nosotros como desarrolladores web debemos cumplir con ciertos estándares para cumplir con la eficiencia, flexibilidad, confiabilidad, mantenibilidad, portabilidad, usabilidad, seguridad e integridad.

Por lo tanto a lo largo de este parcial, vimos lo importantes que son las normas de calidad, que generalmente sirven para evaluar los productos de software, estas contienen métricas, la cuales miden ciertas características de un software, por ejemplo

Métricas orientadas al tamaño: Básicamente es la cantidad de líneas de código y mas características para poder comparar varios proyectos.

Métricas orientadas a la función: El objetivo de esta métrica es medir la funcionalidad del proyecto de software, esta métrica junto con la de orientada al tamaño se puede usar en diferentes técnicas de estimación. La función del software se podrá calcular con PF = cuenta-total x [0.65+0.01 + sum(Fi)]

Personalmente estas métricas son las más completas, además de ser las más usadas junto a otras que determinan los factores ambientales de un proyecto de software. Todo esto nos lleva a las técnicas de estimación de un proyecto de software, aquí también pueden entrar los puntos de función, pero una de las más completas son los puntos de caso de uso.

Técnica de estimación Puntos caso de uso: Esta técnica abarca los casos de uso, actores en un proyecto de software,y factores técnicos y ambientales. Todo esto para calcular las horas totales en esfuerzo de un producto de software y cómo tratamos el factor ambiental también se calcula las horas por persona. Me tome el tiempo de realizar un formato el cual nos da automáticamente el esfuerzo en horas de un proyecto de software: Puntos caso de uso .
