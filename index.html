Proceso de Pruebas: Garantizando la Calidad del Software

prueba es un proceso dinámico que verifica el comportamiento del software mediante un conjunto de casos de prueba

Casos de Prueba: Son escenarios específicos diseñados para evaluar diferentes aspectos del software. Estos casos incluyen entradas, acciones y resultados esperados.

Validación: La validación se centra en determinar si el sistema cumple con los requisitos establecidos. Es decir, ¿el software hace lo que se espera que haga?

Elementos de una Prueba:

a. Valores de Prueba

• Los valores de prueba son las entradas específicas que se utilizan para evaluar el software.

• Por ejemplo, si estamos probando un formulario de registro, los valores de prueba podrían ser nombres, direcciones de correo electrónico, contraseñas, etc.

b. Acciones

• Las acciones son los pasos que se siguen para ejecutar el programa con los valores de prueba.

• Por ejemplo, si estamos probando una función de cálculo de impuestos, la acción podría ser ingresar los ingresos y verificar si el cálculo es correcto.

c. Resultado Esperado

• El resultado esperado es el comportamiento que se espera del software cuando se le proporcionan los valores de prueba.

• Por ejemplo, si estamos probando una aplicación de navegación, el resultado esperado podría ser que muestre la ruta más corta entre dos ubicaciones.
